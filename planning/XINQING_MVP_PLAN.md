# 心晴 - AI 心理健康陪伴应用 MVP 规划

**创建时间**: 2026-02-26  
**项目类型**: iOS 健康应用  
**细分赛道**: 心理健康 + AI 陪伴  
**目标用户**: 25-45 岁城市人群，有压力、焦虑、情绪管理需求

---

## 🎯 产品定位

### 核心价值
**用 AI 技术陪伴用户管理情绪，提供个性化的心理健康支持**

### 差异化优势
1. **AI 陪伴**: 不只是记录，而是主动对话、提供建议
2. **个性化**: 基于用户情绪历史，提供定制化建议
3. **隐私优先**: 所有数据本地存储，不上传云端
4. **简洁易用**: 5 分钟内完成每日情绪记录

---

## 📱 MVP 核心功能（第一阶段）

### 1. 情绪记录（核心功能）

#### 简单记录
- **情绪选择**: 5 种基础情绪（开心、平静、焦虑、难过、愤怒）
- **情绪强度**: 1-10 分评分
- **简短描述**: 可选的文字描述（< 140 字）

#### 快速输入
- **预设标签**: 工作、家庭、感情、健康、金钱、其他
- **语音输入**: 支持语音转文字（iOS 原生）
- **快捷短语**: 常用描述一键输入

#### 数据展示
- **情绪日历**: 日历视图显示每天的情绪
- **趋势图表**: 7 天/30 天情绪趋势
- **情绪分布**: 本周情绪类型占比

### 2. AI 对话（差异化功能）

#### 主动对话
- **每日问候**: 根据用户历史情绪，发送个性化问候
- **情绪关怀**: 检测到负面情绪时，主动询问
- **庆祝时刻**: 检测到积极情绪时，一起庆祝

#### 智能建议
- **呼吸练习**: 当焦虑时，引导 4-7-8 呼吸法
- **正念冥想**: 当压力大时，推荐 5 分钟冥想
- **运动建议**: 根据情绪推荐适合的运动
- **音乐推荐**: 根据情绪推荐放松音乐

#### 对话风格
- **温暖**: 像朋友一样关心
- **不评判**: 接纳所有情绪
- **鼓励**: 给予正向反馈
- **不替做决定**: 引导用户自己思考

### 3. 数据分析（价值功能）

#### 情绪洞察
- **情绪触发词**: 识别常见的情绪触发因素
- **周期性模式**: 发现情绪周期（如每周一焦虑）
- **改善趋势**: 长期使用后的情绪变化趋势

#### 周报
- **每周总结**: 本周情绪概览
- **进步提醒**: 本周比上周好的地方
- **关注提醒**: 需要关注的地方
- **下周建议**: 基于本周数据的建议

## 🛠️ 技术方案

### 前端（SwiftUI）
- **UI 框架**: SwiftUI
- **数据持久化**: CoreData + CloudKit（可选）
- **健康数据**: HealthKit（读取心率、睡眠等）
- **AI 功能**: Core ML（本地模型）+ Claude API（可选）

### AI 方案

#### 初期方案（MVP）
- **本地模型**: 使用苹果原生情绪分析
- **规则引擎**: 基于规则的简单对话
- **模板回复**: 预设的回复模板

#### 后期方案（优化）
- **Claude API**: 调用 Anthropic API 生成对话
- **本地微调**: 基于用户数据微调模型
- **混合方案**: 简单问题本地，复杂问题云端

### 数据模型

```swift
// 情绪记录
struct MoodEntry {
    let id: UUID
    let date: Date
    let mood: MoodType  // 开心、平静、焦虑、难过、愤怒
    let intensity: Int  // 1-10
    let tags: [String]  // 工作、家庭、感情等
    let description: String?
    let aiSuggestion: String?
}

// 情绪类型
enum MoodType: String, CaseIterable {
    case happy = "开心"
    case calm = "平静"
    case anxious = "焦虑"
    case sad = "难过"
    case angry = "愤怒"
}

// AI 对话
struct Conversation {
    let id: UUID
    let date: Date
    let messages: [Message]
}

struct Message {
    let id: UUID
    let content: String
    let isFromUser: Bool
    let timestamp: Date
}
```

## ⚠️ 风险与应对

### 技术风险
- **AI 效果不佳**: 初期使用规则引擎，后期接入 Claude API
- **数据隐私**: 所有数据本地存储，透明隐私政策
- **性能问题**: 优化 CoreData 查询，使用缓存

### 市场风险
- **竞争激烈**: 聚焦 AI 陪伴差异化，不与巨头正面竞争
- **用户留存**: 游戏化设计、每日提醒、情感连接
- **变现困难**: 提供真实价值，免费版功能足够使用

### 合规风险
- **医疗声明**: 明确标注"非医疗诊断工具"
- **数据隐私**: 符合 GDPR、中国网络安全法
- **用户协议**: 清晰的用户协议和隐私政策
